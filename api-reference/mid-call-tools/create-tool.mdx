---
title: "Mid-Call-Tool erstellen"
api: "POST /user/tools"
description: "Neues Mid-Call-Tool erstellen"
---

Dieser Endpunkt ermöglicht es Ihnen, ein neues Mid-Call-Tool zu erstellen, das Ihre KI-Assistenten verwenden können, um während Anrufen mit externen APIs zu interagieren.

### Anfragekörper-Parameter

<ParamField body="name" type="string" required>
  Tool-Name - darf nur Kleinbuchstaben und Unterstriche enthalten und muss mit einem Buchstaben beginnen (z.B. `get_weather`, `book_appointment`)
</ParamField>

<ParamField body="description" type="string" required>
  Detaillierte Erklärung, wann und wie die KI dieses Tool verwenden soll (max. 255 Zeichen)
</ParamField>

<ParamField body="endpoint" type="string" required>
  Gültige URL des aufzurufenden API-Endpunkts
</ParamField>

<ParamField body="method" type="string" required>
  HTTP-Methode: `GET`, `POST`, `PUT`, `PATCH` oder `DELETE`
</ParamField>

<ParamField body="timeout" type="integer" optional>
  Anfrage-Timeout in Sekunden (1-30, Standard: 10)
</ParamField>

<ParamField body="headers" type="array" optional>
  Mit der Anfrage zu sendende HTTP-Header
  <Expandable title="headers-Eigenschaften">
    <ParamField body="name" type="string" required>
      Header-Name
    </ParamField>
    <ParamField body="value" type="string" required>
      Header-Wert
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="schema" type="array" optional>
  Parameter, die die KI aus der Konversation extrahiert und an den Endpunkt sendet
  <Expandable title="schema-Eigenschaften">
    <ParamField body="name" type="string" required>
      Parameter-Name (2-32 Zeichen, muss mit Buchstaben beginnen, kann Buchstaben und Unterstriche enthalten)
    </ParamField>
    <ParamField body="type" type="string" required>
      Parameter-Typ: `string`, `number` oder `boolean`
    </ParamField>
    <ParamField body="description" type="string" required>
      Beschreibung, die der KI hilft zu verstehen, wie dieser Parameter extrahiert werden soll (3-255 Zeichen)
    </ParamField>
  </Expandable>
</ParamField>

### Tools an Assistenten anhängen

Nach dem Erstellen eines Tools müssen Sie es an einen Assistenten anhängen, um es während Anrufen zu verwenden. Tools werden über die Assistant-API verwaltet:

- **[Assistenten erstellen](/api-reference/assistants/create-assistant)** - Verwenden Sie den Parameter `tool_ids`, um Tools beim Erstellen eines Assistenten anzuhängen
- **[Assistenten aktualisieren](/api-reference/assistants/update-assistant)** - Verwenden Sie den Parameter `tool_ids`, um Tools bei einem vorhandenen Assistenten hinzuzufügen, zu entfernen oder zu ersetzen
