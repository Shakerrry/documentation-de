---
title: "Eingehenden Assistenten-Webhook deaktivieren"
api: "POST /user/assistants/disable-inbound-webhook"
description: "Eingehende Webhook-Benachrichtigungen für einen bestimmten Assistenten deaktivieren"
---

Dieser Endpunkt deaktiviert eingehende Webhook-Benachrichtigungen für einen Assistenten und stoppt die Zustellung von Echtzeit-Updates über eingehende Anrufabschlüsse und Daten.

### Anfragekörper

<ParamField body="assistant_id" type="integer" required>
  Die ID des Assistenten, für den der Webhook deaktiviert werden soll
</ParamField>

### Antwort

<ResponseField name="message" type="string">
  Erfolgsmeldung, die bestätigt, dass der Webhook deaktiviert wurde
</ResponseField>

<ResponseField name="data" type="array">
  Leeres Array (für zukünftige Verwendung reserviert)
</ResponseField>

### Fehlerantworten

<ResponseField name="404 Not Found">
  <Expandable title="Fehlerantwort">
    <ResponseField name="message" type="string">
      Fehlermeldung, wenn der Assistent nicht gefunden wurde oder dem authentifizierten Benutzer nicht gehört
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="422 Validation Error">
  <Expandable title="Fehlerantwort">
    <ResponseField name="message" type="string">
      Fehlermeldung, die einen Validierungsfehler anzeigt
    </ResponseField>
    <ResponseField name="errors" type="object">
      Detaillierte Validierungsfehler für jedes Feld
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

```json 200 Erfolgsantwort
{
  "message": "Inbound webhook disabled successfully",
  "data": []
}
```

```json 404 Not Found
{
  "message": "Assistant not found"
}
```

```json 422 Validierungsfehler
{
  "message": "The given data was invalid.",
  "errors": {
    "assistant_id": [
      "The assistant id field is required."
    ]
  }
}
```

</ResponseExample>
