---
title: "Assistenten löschen"
api: "DELETE /user/assistant/{id}"
description: "Einen bestimmten KI-Assistenten löschen"
---

Dieser Endpunkt ermöglicht es Ihnen, einen KI-Assistenten, der dem authentifizierten Benutzer gehört, dauerhaft zu löschen.

### Pfad-Parameter

<ParamField path="id" type="integer" required>
  Die eindeutige Kennung des zu löschenden Assistenten
</ParamField>

### Antwort

<ResponseField name="message" type="string">
  Bestätigungsnachricht, die anzeigt, dass der Assistent erfolgreich gelöscht wurde
</ResponseField>

### Fehlerantworten

<ResponseField name="404 Not Found">
  <Expandable title="Fehlerantwort">
    <ResponseField name="message" type="string">
      Fehlermeldung, wenn der Assistent nicht gefunden wurde oder dem authentifizierten Benutzer nicht gehört
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="500 Internal Server Error">
  <Expandable title="Fehlerantwort">
    <ResponseField name="message" type="string">
      Fehlermeldung, die das Fehlschlagen der Assistentenlöschung anzeigt
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

```json 200 Erfolgsantwort
{
  "message": "Assistant deleted successfully"
}
```

```json 404 Not Found
{
  "message": "Assistant not found"
}
```

```json 500 Internal Server Error
{
  "message": "Assistant deletion failed"
}
```

</ResponseExample>

### Hinweise

- Nur Assistenten, die dem authentifizierten Benutzer gehören, können gelöscht werden
- Sobald ein Assistent gelöscht wurde, kann er nicht wiederhergestellt werden
- Diese Aktion entfernt den Assistenten und seine gesamte Konfiguration dauerhaft
- Wenn der Assistent derzeit aktive Anrufe bearbeitet, können diese Anrufe beeinträchtigt werden
