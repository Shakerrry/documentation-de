## Was ist SIP-Integration?

SIP-Integration (Session Initiation Protocol) ermöglicht es Ihnen, Ihr bestehendes Telefonsystem mit Website Excellence Builders.ai zu verbinden. Stellen Sie es sich vor wie eine Brücke zwischen Ihrem aktuellen Telefon-Setup und unserer KI-Plattform. Das bedeutet, Sie können:

- Ihre bestehenden Telefonnummern mit unseren KI-Assistenten verwenden
- Ihr aktuelles Telefonsystem und Ihren Anbieter behalten
- Anrufe über Ihr bestehendes Setup tätigen und empfangen
- Volle Kontrolle über Ihre Nummern behalten
- Carrier-Kosten reduzieren, indem Sie lokale VoIP-Anbieter mit besseren Tarifen nutzen

## Wie es funktioniert

Wenn Sie Ihr Telefonsystem mit Website Excellence Builders.ai integrieren:

1. **Eingehende Anrufe**: Anrufe an Ihre Nummern werden über Ihr Telefonsystem an unsere KI-Assistenten weitergeleitet
2. **Ausgehende Anrufe**: Unsere KI-Assistenten können Anrufe tätigen, die von Ihren Nummern zu stammen scheinen
3. **Verwaltung**: Sie verwalten Ihre Nummern weiterhin über Ihren aktuellen Anbieter
4. **Kosteneffizienz**: Nutzen Sie die wettbewerbsfähigen Tarife Ihres VoIP-Anbieters für die Anrufbeendigung

## Integrationsoptionen

Sie können Ihr Telefonsystem auf zwei Arten mit Website Excellence Builders.ai verbinden:

### 1. PBX-Nebenstellen-Integration

Das ist wie unserer KI-Plattform eine "Nebenstelle" auf Ihrem Telefonsystem zu geben:

- **Was es ist**: Verbinden Sie eine bestimmte Nebenstelle von Ihrer PBX/Ihrem Telefonsystem
- **Perfekt für**:
  - Testen der Integration
  - Weiterleitung bestimmter Nebenstellen zur KI
  - Einfache Einrichtung mit bestehenden Telefonsystemen
- **Sie benötigen**:
  - Ihre Nebenstellennummer (z.B. 1000)
  - Nebenstellen-Benutzername und Passwort
  - Ihre PBX-Serveradresse (z.B. pbx.ihrefirma.com)

**Reales Beispiel**:

```
Ihr Büro verwendet Nebenstellen 1001-1010 für Mitarbeiter
Sie richten Nebenstelle 1011 für den KI-Assistenten ein
Anrufe an 1011 gehen zur KI
Die KI kann Anrufe tätigen, die als Nebenstelle 1011 erscheinen
```

### 2. Direkte Nummern-Integration

Dies verbindet bestimmte Telefonnummern mit unserer Plattform:

- **Was es ist**: Leiten Sie eine oder mehrere Ihrer Telefonnummern zu unserer KI weiter
- **Perfekt für**:
  - Dedizierte KI-Telefonleitungen
  - Kundenservice-Nummern
  - Vertriebs- oder Support-Leitungen
- **Sie benötigen**:
  - Ihre Telefonnummer (z.B. +1 555 0123)
  - SIP-Anmeldedaten von Ihrem Anbieter
  - SIP-Serveradresse des Anbieters

**Reales Beispiel**:

```
Sie haben eine Support-Nummer: +1 555 0123
Sie verbinden diese Nummer mit unserer KI
Alle Anrufe an +1 555 0123 werden vom KI-Assistenten bearbeitet
Die KI tätigt ausgehende Anrufe mit +1 555 0123
```

## Authentifizierung & Anruffluss

### Bidirektionale Konfiguration

Die SIP-Integration verwendet die gleichen Anmeldedaten für sowohl eingehende als auch ausgehende Anrufe:

- **Eingehende Anrufe** (VoIP → Website Excellence Builders.ai):

  ```
  Ihr VoIP-System authentifiziert sich mit Benutzername/Passwort
  ↓
  Anrufe erreichen unsere Plattform
  ↓
  KI-Assistent antwortet
  ```

- **Ausgehende Anrufe** (Website Excellence Builders.ai → VoIP):
  ```
  KI initiiert Anruf
  ↓
  Unsere Plattform authentifiziert sich bei Ihrem VoIP mit denselben Benutzername/Passwort
  ↓
  Anruf erreicht Ziel
  ```

### Anmeldedaten-Synchronisation

Damit die Integration ordnungsgemäß funktioniert:

1. Die Anmeldedaten, die Sie im Website Excellence Builders.ai-Dashboard festlegen, müssen exakt mit Ihrer VoIP-Konfiguration übereinstimmen
2. Die gleiche Benutzername/Passwort-Kombination wird für beide Richtungen verwendet
3. Alle Änderungen an den Anmeldedaten müssen in beiden Systemen aktualisiert werden

<Warning>
  Wenn Sie Anmeldedaten in Ihrem VoIP-System ändern, denken Sie daran, sie im Website Excellence Builders.ai-Dashboard zu aktualisieren (und umgekehrt), um die Verbindung aufrechtzuerhalten.
</Warning>

## Ihre Integration einrichten

### Schritt 1: Sammeln Sie Ihre Informationen

Für PBX-Nebenstelle:

```
Nebenstellennummer: z.B. 1000
Serveradresse: z.B. pbx.firma.com
Benutzername: Ihr Nebenstellen-Benutzername
Passwort: Ihr Nebenstellen-Passwort
```

Für Direkte Nummer:

```
Telefonnummer: z.B. +1 555 0123
SIP-Server: z.B. sip.anbieter.com
Benutzername: Von Ihrer Telefongesellschaft bereitgestellt
Passwort: Von Ihrer Telefongesellschaft bereitgestellt
```

### Schritt 2: Im Dashboard konfigurieren

1. Gehen Sie zu Ihrem Website Excellence Builders.ai-Dashboard
2. Navigieren Sie zu **Ihre Telefonnummern**
3. Klicken Sie auf "SIP-Trunk integrieren"

### Schritt 3: SIP-Trunk-Typ auswählen

Wählen Sie oben im Formular einen dieser Trunk-Typen aus:

- **SIP-Nebenstelle** - Verwenden Sie dies, wenn Sie bereits eine vorhandene SIP-Anmeldedaten (eine Nebenstelle) von Ihrem VoIP-Anbieter erhalten haben.
- **Telefonnummer (DID)** - Verwenden Sie dies, wenn Sie eine dedizierte Direct Inward Dial (DID)-Nummer anstelle einer generischen SIP-Nebenstelle integrieren möchten.

Für PBX-Nebenstelle:
```
Nebenstellennummer: z.B. 1000
Benutzername: Ihr Nebenstellen-Benutzername
Passwort: Ihr Nebenstellen-Passwort
```

Für Direkte Nummer:

```
Telefonnummer: z.B. +1 555 0123
Benutzername: Von Ihrer Telefongesellschaft bereitgestellt
Passwort: Von Ihrer Telefongesellschaft bereitgestellt
```
<img height="200" src="/resources/sip-integration-did.png" />

### Schritt 4: Ausgehende Einstellungen

Diese Einstellungen steuern, wie Website Excellence Builders.ai ausgehende Anrufe über Ihren SIP-Anbieter sendet.

- **SIP-Adresse** - Geben Sie die Adresse/den Server des SIP-Trunks ohne Port ein.
- **Outbound-Proxy (Feste IP)** - Einige SIP-Anbieter verlangen, dass Anrufe von einer einzelnen, auf der Whitelist stehenden IP-Adresse stammen. Konfigurieren Sie dies wie folgt:
  - **EIN schalten** - Wenn Ihr SIP-Anbieter eine feste Quell-IP benötigt, aktivieren Sie den Outbound-Proxy, der von einer festen IP-Adresse kommt. Website Excellence Builders.ai sendet dann alle ausgehenden Anfragen über eine bekannte statische IP.
  Verwenden Sie diese Option nur, wenn Ihr Anbieter explizit eine einzelne Quell-IP verlangt.
  - **AUS schalten:** - Wenn Ihr Anbieter Anrufe von jeder IP akzeptiert, lassen Sie diese Option deaktiviert. In diesem Fall sendet Website Excellence Builders.ai Anrufe von dynamischen IPs.

```
SIP-Adresse: (z.B. pbx.firma.com/sip.anbieter.com)
```
<img height="200" src="/resources/sip-integration-outbound.png" />

### Schritt 5: Nummernformat-Konfiguration

Dies teilt unserem System mit, wie Telefonnummern beim Tätigen von Anrufen formatiert werden sollen. Wählen Sie basierend auf den Anforderungen Ihres Anbieters:

1. **International mit +**

   ```
   Format: +[Land][Nummer]
   Beispiel: +1 202 555 0123
   SIP-Anfrage: sip:+12025550123@ihre.pbx.com
   ```

2. **International ohne +**

   ```
   Format: [Land][Nummer]
   Beispiel: 1 202 555 0123
   SIP-Anfrage: sip:12025550123@ihre.pbx.com
   ```

3. **Nationales Format**
   ```
   Format: [Lokale Nummer]
   Beispiel: 0741926265
   SIP-Anfrage: sip:0741926265@ihre.pbx.com
   ```

### Schritt 6: Eingehende Einstellungen

Diese Einstellungen bestimmen, wie Website Excellence Builders.ai eingehende Anrufe von Ihrem SIP-Anbieter empfängt und authentifiziert.

1. **Unsere SIP-Adresse** - Website Excellence Builders.ai weist eine eindeutige eingehende SIP-URI zu ```5wed79aeavr.sip.livekit.cloud```. Ihr SIP-Anbieter muss so konfiguriert sein, dass er eingehende Anrufe an diese Adresse sendet, indem er seinen ausgehenden Trunk oder seine Routing-Regel auf diese URI verweist.
2. **IP-Adresse (Whitelist-IPs)** - Wenn Ihr SIP-Anbieter Anrufe von einem bekannten Satz öffentlicher IP-Adressen senden kann, wählen Sie IP-Adresse. Fügen Sie dann jede Anbieter-IP zur Whitelist unter Eingehende Authentifizierung hinzu.
3. **Benutzername und Passwort** - Wenn Ihr Anbieter SIP-Authentifizierung unterstützt, wählen Sie Benutzername und Passwort. Die gleichen Anmeldedaten, die Sie unter Ausgehende Einstellungen eingegeben haben (Benutzername/Passwort), werden zur Authentifizierung eingehender Anrufe verwendet. Stellen Sie sicher, dass die PBX Ihres Anbieters so konfiguriert ist, dass sie diese Anmeldedaten enthält.

### Schritt 7: Länderauswahl
Wählen Sie das Land aus, in dem Ihr SIP-Trunk bereitgestellt ist.

### Schritt 8: Einrichtung überprüfen

Nach der Konfiguration:

1. Tätigen Sie einen Testanruf an Ihre Nummer/Nebenstelle
2. Überprüfen Sie, dass der KI-Assistent antwortet
3. Verwenden Sie die Testanruf-Funktion, um ausgehende Anrufe zu überprüfen

## Sicherheitsanforderungen

Ihre SIP-Anmeldedaten müssen sicher sein:

- **Benutzername**: Ihre Nebenstellennummer oder vom Anbieter bereitgestellter Benutzername
- **Passwort muss haben**:
  - 12 oder mehr Zeichen
  - Zahlen
  - Groß- und Kleinbuchstaben
  - Sonderzeichen empfohlen
