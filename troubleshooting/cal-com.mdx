Lösungen für häufige Probleme mit der Cal.com-Integration.

## Verbindungsprobleme

### Ungültiger API-Schlüssel

**Problem:** Events werden nicht geladen oder Sync schlägt fehl

**Lösung:**
- Überprüfen Sie, ob der API-Schlüssel in Cal.com → Einstellungen → Entwickler → API-Schlüssel aktiv ist
- Stellen Sie sicher, dass der Schlüssel das Präfix `cal_live_` enthält
- Generieren Sie bei Bedarf einen neuen API-Schlüssel

### Events werden nicht geladen

**Problem:** Event-Dropdown ist leer nach Eingabe des API-Schlüssels

**Lösung:**
- Überprüfen Sie, ob der API-Schlüssel korrekt ist
- Klicken Sie außerhalb des API-Schlüssel-Feldes, um das Laden der Events auszulösen
- Aktualisieren Sie die Seite, wenn Events immer noch nicht erscheinen
- Überprüfen Sie, ob Sie Events in Cal.com erstellt haben

## Synchronisierungsprobleme

### Sync Event fehlgeschlagen

**Problem:** "Sync Event"-Button schlägt fehl oder zeigt Fehler an

**Lösung:**
1. Klicken Sie stattdessen auf den **"Troubleshoot"**-Button
2. Dies setzt alle Buchungsfelder auf die korrekte Konfiguration zurück
3. Überprüfen Sie die Änderungen in den Cal.com-Event-Einstellungen

### Feldkonfigurationsfehler

**Problem:** Fehler bezüglich erforderlicher Felder oder fehlender Felder

**Lösung:**
- Klicken Sie auf **"Troubleshoot"**, um Felder zurückzusetzen
- Nur Name, E-Mail und Telefon sollten erforderlich sein
- Alle anderen Felder (Standort, Gäste usw.) sollten optional sein

## Buchungsfehler

### Keine Kalendereinladungen gesendet

**Problem:** Buchungen sind erfolgreich, aber der Kunde erhält keine Einladung

**Lösung:**
- Definieren Sie die Variable `email` in den [Assistenteneinstellungen](/ai-assistants/settings/prompt-and-tools#call-variables)
- Übergeben Sie die E-Mail-Adresse des Kunden beim Erstellen von Leads
- Überprüfen Sie, ob das E-Mail-Format gültig ist

### Telefonnummer fehlt

**Problem:** Telefonnummer erscheint nicht in Kalenderereignissen

**Lösung:**
- Klicken Sie auf den **"Sync Event"**- oder **"Troubleshoot"**-Button
- Dadurch wird das Telefonfeld automatisch zum Event hinzugefügt
- Telefonnummer wird nach der Synchronisierung einbezogen

### Probleme mit der Buchungszeitzone

**Problem:** Termine werden zur falschen Zeit geplant

**Lösung:**
- Überprüfen Sie die Zeitzone in den Cal.com-Event-Einstellungen
- Überprüfen Sie, ob der Assistent die korrekte Zeit vom Kunden erhält
- Testen Sie mit spezifischen Zeitangaben während des Anrufs

## Testprobleme

### Web-Anruf-Buchungen schlagen fehl

**Problem:** Buchungen funktionieren inkonsistent mit Web-Anrufen

**Lösung:**
- Verwenden Sie echte Telefonanrufe zum Testen (keine Web-Anrufe)
- Web-Anrufe geben möglicherweise die Telefonvariable nicht korrekt weiter
- Geben Sie bei Web-Anrufen die Telefonnummer im internationalen Format mit +-Präfix an (z. B. "+1 555-123-4567")
- Telefonanrufe stellen sicher, dass alle Variablen korrekt übertragen werden

### Mehrere Kalender funktionieren nicht

**Problem:** Nur ein Kalender funktioniert oder Events kollidieren

**Lösung:**
- Jeder Kalender sollte ein anderes ausgewähltes Event haben
- Verwenden Sie eindeutige Event-Namen (z. B. "15min", "30min")
- Synchronisieren Sie jedes Event separat nach dem Hinzufügen

## Hilfe erhalten

Wenn Probleme weiterhin bestehen:

1. Überprüfen Sie Anruftranskripte auf Buchungsversuche
2. Überprüfen Sie, ob das Cal.com-Event im Cal.com-Dashboard angezeigt wird
3. Testen Sie mit einer einfachen Telefonbuchung
4. Kontaktieren Sie den Support mit:
   - Screenshot der Fehlermeldung
   - Screenshot der Cal.com-Event-Einstellungen
   - Anruftranskript, das den Buchungsversuch zeigt
