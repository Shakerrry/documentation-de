Dieser Leitfaden hilft bei der Lösung häufiger SIP-Integrationsprobleme beim Verbinden Ihres SIP-Anbieters mit Autocalls.ai.

## Teststrategie

1. **Ausgehende Anrufe zuerst testen** - einfacher zu konfigurieren
2. **Eingehende Anrufe zweitens testen** - nachdem ausgehende funktionieren

## Probleme mit ausgehendem SIP

### Verbindungsprobleme

**SIP-Trunk verbindet sich nicht für ausgehende Anrufe:**

1. **SIP-Anmeldedaten überprüfen** - Benutzername, Passwort, Serverdetails
2. **Telefontypauswahl** - mit Anbieter auf korrekten Typ prüfen
3. **Feste IP-Konfiguration** - normalerweise aktivieren und IP-Adresse des Anbieters eingeben
4. **Anbietereinstellungen** - Codec, Transport, Port-Einstellungen überprüfen

### SIP-Registrierungsprobleme

**Wichtig:** Autocalls.ai-Plattform:
- **Sendet KEINE SIP REGISTER**-Nachrichten an Ihren Anbieter
- **Benötigt KEINEN Empfang von SIP REGISTER** von Ihrem Anbieter
- Verwendet stattdessen **direkte Anrufauthentifizierung**

### Ausgehende Tests

1. SIP-Trunk-Einstellungen konfigurieren
2. **Direkten Testanruf** über Plattform durchführen
3. Anrufprotokolle auf Fehler überprüfen

## Probleme mit eingehendem SIP

### Authentifizierungs-Setup

**Für eingehende Anrufe:**

1. **IP-Authentifizierung verwenden** - anstelle von Benutzername/Passwort
2. **Anbieter-IPs auf Whitelist setzen** - ALLE Anbieter-IPs in unserer Plattform hinzufügen
3. **Anbieterkonfiguration - WICHTIG**
   - **Unsere FQDN-Domain verwenden** für Routing (NICHT die ausgehende IP)
   - **An Domain senden** - stellt korrektes Routing sicher
   - **NICHT an IP senden** - verursacht Routing-Probleme

### Häufige Probleme mit eingehenden Anrufen

**Anrufe erreichen Assistenten nicht:**
- Alle Anbieter-IPs auf Whitelist überprüfen
- Sicherstellen, dass Anbieter an FQDN-Domain sendet (nicht IP)
- Zielnummernformat überprüfen

## Allgemeine Probleme

**Netzwerkprobleme:**
- Internetverbindung überprüfen
- Überprüfen, ob keine Firewall SIP-Traffic blockiert

**Anbieterkompatibilität:**
- Überprüfen, ob Anbieter direkte Anrufe unterstützt (keine Registrierung)
- IP-Authentifizierungsunterstützung prüfen

## Test-Checkliste

### Vor Kontaktaufnahme mit Support

1. **Ausgehender Test:**
   - [ ] Anmeldedaten überprüft
   - [ ] Telefontyp korrekt ausgewählt
   - [ ] Feste IP mit IP-Adresse des Anbieters aktiviert
   - [ ] Direkter Testanruf versucht

2. **Eingehender Test:**
   - [ ] IP-Authentifizierung konfiguriert
   - [ ] Alle Anbieter-IPs in unserer Plattform auf Whitelist gesetzt
   - [ ] Anbieter konfiguriert, um an FQDN-Domain zu senden (nicht direkte IP)
   - [ ] Testanruf von externer Nummer

3. **Allgemein:**
   - [ ] Netzwerkkonnektivität überprüft

## FAQ

### Telefonanrufweiterleitung funktioniert nicht

**F: Warum funktioniert die Telefonanrufweiterleitung nicht?**

**A:** Wenn Anrufweiterleitungen fehlschlagen, versuchen Sie diese Lösungen:

1. **SIP-Transfer-Einstellungen überprüfen:**
   - Transfer-Anruf auf "Erweitert"-Modus setzen
   - SIP-URI-Format verwenden: `sip:nummer@sip-server`
   - Beispiel: `sip:+1234567890@sip.provider.com:5060`

2. **Anbieterunterstützung überprüfen:**
   - Bestätigen Sie, dass Ihr SIP-Anbieter SIP REFER unterstützt
   - Prüfen Sie, ob Transfer-Funktionalität auf Ihrem Konto aktiviert ist
   - Einige Anbieter erfordern spezifische Konfiguration für Transfers

3. **Verschiedene Formate testen:**
   - Mit Port versuchen: `sip:+1234567890@sip-server:5060`
   - Ohne Port versuchen: `sip:+1234567890@sip-server`
   - Einfaches Format versuchen: `sip:+1234567890`

4. **Ziel überprüfen:**
   - Sicherstellen, dass die Zielnummer erreichbar ist
   - Testen, ob die Nummer Anrufe normal beantwortet
   - Prüfen, ob die Nummer blockiert oder eingeschränkt ist

5. **Häufige Probleme:**
   - Anbieter unterstützt SIP REFER nicht
   - Falsches SIP-URI-Format
   - Netzwerk/Firewall blockiert Transfer-Anfragen
   - Zielnummer antwortet nicht

**Fehlerbehebungsschritte:**
1. Zuerst mit einer bekannt funktionierenden Nummer testen
2. Verschiedene SIP-URI-Formate ausprobieren
3. Ihren SIP-Anbieter kontaktieren, um Transfer-Unterstützung zu bestätigen
4. Anrufprotokolle auf spezifische Fehlermeldungen prüfen
